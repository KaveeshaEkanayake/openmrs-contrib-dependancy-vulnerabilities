# OpenMRS Dependency Vulnerability Dashboard

A production-quality React dashboard that displays dependency vulnerability reports for OpenMRS modules. This tool helps maintainers visualize and prioritize security vulnerabilities across multiple OpenMRS projects.

## Features

- **Repository-Level Overview**: View vulnerabilities grouped by OpenMRS module
- **Dependency Details**: See affected dependencies with versions and severity levels
- **CVE Information**: Expandable CVE details including descriptions, scores, and fix versions
- **Severity Badges**: Color-coded badges for Critical, High, Medium, and Low severities
- **Interactive Sorting**: Sort repositories by Severity, CVE Count, or Name (A-Z)
- **Exploit Detection**: Identifies vulnerabilities with known exploits
- **Keyboard Accessible**: Full keyboard navigation support (Tab, Enter, Space)
- **Responsive Design**: Clean, professional UI that works across devices
- **No Backend Required**: All data is processed client-side from JSON files

## Tech Stack

- **React 18** - Modern React with functional components and hooks
- **Plain CSS** - No heavy UI frameworks, just clean custom styles
- **OWASP Dependency-Check** - Source data format

## Getting Started

### Prerequisites

- Node.js 14+ and npm

### Installation

1. Clone the repository:
```bash
git clone https://github.com/jayasanka-sack/openmrs-contrib-dependancy-vulnerabilities.git
cd openmrs-contrib-dependancy-vulnerabilities
```

2. Install dependencies:
```bash
npm install
```

3. Start the development server:
```bash
npm start
```

4. Open [http://localhost:3000](http://localhost:3000) in your browser

### Building for Production

```bash
npm run build
```

This creates an optimized production build in the `build/` directory.

## Data Source

The dashboard reads vulnerability reports from JSON files in `public/data/`:

- `openmrs-core.json`
- `openmrs-module-billing.json`
- `openmrs-module-idgen.json`

These files are generated by [OWASP Dependency-Check](https://github.com/jeremylong/DependencyCheck) and follow the GitLab Security Report schema.

### Adding New Reports

To add a new module:

1. Place the JSON file in `public/data/`
2. Update the repository list in `src/hooks/useVulnerabilityData.js`:

```javascript
const repositories = [
  'openmrs-core',
  'openmrs-module-billing',
  'openmrs-module-idgen',
  'your-new-module', // Add here
];
```

## Key Components

### useVulnerabilityData Hook

Fetches and processes vulnerability data:
- Groups vulnerabilities by dependency
- Calculates severity levels from CVE data
- Detects exploits from descriptions and reference links
- Extracts fix versions from CVE descriptions
- Sorts dependencies and CVEs by severity (highest first)

### Severity Calculation

Dependencies inherit the highest severity from their CVEs:
- Critical: 9.0+
- High: 7.0-8.9
- Medium: 4.0-6.9
- Low: 0.1-3.9

### Version Comparison

Semantic version comparison to determine the highest fix version across all CVEs in a dependency.
